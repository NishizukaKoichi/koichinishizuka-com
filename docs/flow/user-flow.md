# User Flow Definition

必須項目: 以下は埋め済み。未記入のまま次工程へ進まない。

## entry_channels
自分で作ったアプリやスクリプトを「売れる形」で配布したいタイミングで入る

## first_success
実行・配布・停止の判断が entitlement に一本化され、
「これは売っていい」「これは止められる」という状態が仕組みとして保証される

## payment_consideration
課金・権利管理・配布制御を毎回ゼロから考えるのが重くて手が止まっている

## post_purchase
課金や配布の設計を考えずに“作ること”に戻れる
