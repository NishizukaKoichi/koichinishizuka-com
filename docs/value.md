# MagicSpell Value Definition

必須項目: 以下は埋め済み。未記入のまま次工程へ進まない。

## who
個人開発者・小規模SaaS運営者で、複数の小さなプロダクトやツールを並行して扱っている人

## situation
自分で作ったアプリやスクリプトを「売れる形」で配布したいが、
課金・権利管理・配布制御を毎回ゼロから考えるのが重くて手が止まっている

## pain
Stripeや認証や配布を個別実装すると、
・設計が毎回ブレる
・売り方を変えたくなった瞬間に壊れる
・「ちゃんと止まるか」「剥奪できるか」が信用できない
結果として、作ったものを安心して売れない

## after_paying
実行・配布・停止の判断がすべて entitlement に一本化され、
「これは売っていい」「これは止められる」という状態が
仕組みとして保証されるので、
課金や配布の設計を考えずに“作ること”に戻れる
